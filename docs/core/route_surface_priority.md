# Route Surface Priority

`habit` route は、低刺激の入力に対する軽い即応として残す。
ただし、次のような **構造化された Inner OS の表出優先** が立っているターンでは、
`habit` を優先しない。

- `green_reflection_hold`
- `reflect_only` / `boundary_only` の contact reflection
- `reopen_from_anchor` 系の thread reopening
- deep disclosure の reflection act

この制約により、deep talk や継続中の話題で generic な
`どこから続けますか` / `もちろんです` 系の habit fallback が
最終表出を上書きしにくくなる。

要点は、route の簡便さより **そのターンで既に成立している内的 readout を守る** ことにある。
