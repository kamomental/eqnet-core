# EQNet Core Emotional DFT v0.1

## 目的
この文書は、2026-03-15 の議論で固まった

- 感情地形
- グリーン関数
- 主観アクセス界面
- Emotional DFT

を、`Inner OS` に実装可能な形へ落とした最小仕様である。

ここで重要なのは、LLM を知能本体にしないことだ。  
LLM は表出器であり、状態更新・価値制御・アクセス選択は構造化された内部状態系で担う。

## 核となる分離
本モデルは次の 4 役割を分ける。

1. グリーン関数: 刺激がいつどう響くか
2. 感情地形: 状態がどこへ流れるか
3. 主観アクセス界面: 何が前景として触れるか
4. Emotional DFT: それらの動的構造をどう分解して読むか

この 4 つを混ぜると理論が曖昧になるので、役割分離を守る。

## 1. 力学コア
内部状態を `x(t) ∈ R^n` とし、最小の力学コアを次で置く。

```text
dx/dt = -∇Φ(x, t) + B u(t) - Γ L_G x + ξ(t)
```

各項の意味は次の通り。

- `Φ(x, t)`: 感情地形、価値ポテンシャル
- `u(t)`: 外部刺激
- `B`: 刺激の入力結合
- `L_G`: 関係グラフのラプラシアン
- `Γ`: 関係伝播の強さ
- `ξ(t)`: ノイズ、身体ゆらぎ、未モデル化変動

この力学コアは、主観仮説が弱かったとしても残すべきコアである。

## 2. 驚き
予測誤差を

```text
e(t) = y(t) - ŷ(t)
```

とし、驚き `S(t)` は予測外れ、地形勾配、局所曲率の合成で見る。

```text
S(t) = α ||e(t)|| + β ||∇Φ(x,t)|| + γ λ_max(H_Φ(x,t))
```

ここで

- `||e(t)||`: 予測外れの大きさ
- `||∇Φ(x,t)||`: いまの状態が流される強さ
- `λ_max(H_Φ)`: 局所曲率の鋭さ

である。

注意すべき点は、`prediction error = |∇Φ|` と同一視しないことだ。  
予測誤差は地形を変形しうる入力であり、地形勾配そのものとは役割が違う。

## 3. グリーン関数と慣れ
平衡点近傍で線形化できるとき、応答核はグリーン関数で書ける。

```text
δx(t) = ∫ G(t-τ) B u(τ) dτ
```

最小近似では

```text
G(t) ≈ exp(-λt)
```

のような減衰核を使える。

ただし慣れは単なる核の減衰だけでは足りない。最低でも次の 2 層に分ける。

1. 応答核の減衰  
   同じ刺激への即時反応が弱くなる
2. 地形の平滑化  
   繰り返し体験により、その刺激周辺の崖や盆地が変形する

したがって、慣れは

```text
habituation = kernel_decay + terrain_reshaping
```

として扱う。

## 4. 主観アクセス界面
主観経験は `f(x)` の一点値ではなく、アクセス様式に依存する追加層として扱う。

```text
I ≠ f(x) のみ
I = f(x, access_mode)
```

この `access_mode` を幾何学的に近似したものが主観アクセス界面 `Q_t` である。

```text
Q_t : R^k → R^n
```

ここで `k=2` は v0.1 の PoC 用近似であって、本質的公理ではない。  
現時点で本質なのは「低次元連続界面」であり、2 次元固定は仮説に留める。

主観強度の最小像は

```text
I_t(u) = Φ(Q_t(u), t)
```

で書けるが、実装ではまず `access_mode` を reportable / controllable / memory-relevant な前景選択として扱えばよい。

## 5. Emotional DFT
ここでの DFT は `Dynamic Field Transform` として定義する。  
目的は、感情ダイナミクスを交換可能な解析層として読むことにある。

少なくとも次の 3 分解を扱えるようにする。

1. 時間モード  
   速い驚き、尾を引く不安、反芻のような遅いモード
2. 関係グラフモード  
   感情が全体に広がっているか、特定関係に局在しているか
3. 力学モード  
   どの安定盆地や遷移経路に乗っているか

この層はコア力学ではなく、比較可能な解析層として保つ。

## 6. コアと交換可能層
理論を守る対象を分ける。

### Core
- 状態方程式
- 感情地形
- 刺激入力
- 関係グラフ結合
- 驚き / 記憶固定 / 散逸の評価量

### Replaceable
- 主観モデル
- 主観アクセス界面の形
- DFT / Koopman / wavelet / graph spectral の解析層

### Experimental
- 2 次元固定の必然性
- 界面特異点と解離の対応
- 強いクオリア主張

## 7. 主観仮説群
主観層は比較可能な仮説群として扱う。

### M0: State only
```text
I = f(x)
```

### M1: Attention weighted
```text
I = f(x, attention)
```

### M2: Access mode
```text
I = f(x, access_mode)
```

### M3: Geometry interaction
```text
I = f(x, interface_geometry, attention)
```

M3 を最初から勝たせるのではなく、M1/M2 より識別力が上がるかを検証する。

## 8. 反証条件
この理論は次の条件で弱まる。

1. `M1` や `M2` が `M3` と同等以上の識別性能を示す
2. 界面幾何の変更が主観推定に系統的差を生まない
3. 関係結合 `Γ` やシナリオ変化に対して `M3` が追加説明力を示さない

## 9. Inner OS への接続
`Inner OS` では次の責務分離で扱う。

- Grounding: 観測 → アフォーダンス → 記号接地
- State Core: world / self / continuity
- Value & Access: 感情地形、attention、foreground selection
- Expression Bridge: LLM は foreground のみを見る

つまり、`Emotional DFT` は `Inner OS` の value / access 層を厚くする理論であって、LLM 層を肥大化させる理論ではない。

## 10. v0.1 の立場
v0.1 は完成理論ではない。  
あくまで

- 力学コアは強く固定する
- 主観層は交換可能仮説として扱う
- LLM は表出層に留める

という研究 OS の姿勢を明示した版である。
