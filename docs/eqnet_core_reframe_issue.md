# EQNet Core 再構成イシュー（Draft）

## 0. きっかけ / Context
`agi-memory` に "heartbeat" があり、言葉は近いが目的は別だと感じた。  
あちらは「メモリの管理」、こちらは「行動を安定させる制御ループ」。  
重なるのは語彙であり、狙いは違う。

このドキュメントは以下を兼ねる:
- **イシュー**: 何を決め、何を作るかの整理
- **説明文**: 一般にも通じる EQNet Core の要約

---

## 1. 課題（なぜ再構成するか）
現状の説明は「概念」と「実装」が混ざっていて、初見の人が以下を想像しにくい。
- 何を目指しているのか（ゴール、ユースケース、価値）
- 既存の「プロンプト調整 + 監査」で代替できそうに見える
- heartbeat / memory など似た語があり、誤解されやすい

**EQNet Core の狙いと範囲を再構成し、わかりやすく言語化する**。

---

## 2. EQNet Core とは（短い説明）
EQNet Core は **人と関わる AI の行動を安定させる制御ループ**。  
AGI ではなく、**長期の対話・関係の中で安全かつ一貫性ある振る舞い**を作る仕組み。

LLM を内蔵するのではなく、LLM の外側に「振る舞いの調整層」を置く:
- **状態を測る**（緊張、共鳴、リスク、テンポ）
- **出力を調整する**（速度、トーン、介入・抑制）
- **夜間監査で修正する**（ドリフト検知、復旧）

---

## 3. これは何ではないか
### AGI / ASI ではない
- **AGI** は広い問題解決能力を目指す
- **ASI** は超人的な能力を目指す
- **EQNet Core** は「能力」ではなく「行動の安定性」を狙う

### プロンプト調整ではない
プロンプト調整は「1回の応答」を微調整する。  
EQNet Core は「継続的な振る舞い」を制御する。

---

## 4. よくある反論への整理
### 「複数エージェントの監査や MCP でできるのでは？」
監査やルールは「検知・制約」には有効だが、**連続的な挙動の制御**ではない。

EQNet Core は:
- 状態変数と時間定数を持つ
- 記憶階層を跨いで振る舞いを整える
- 「検知」ではなく「舵取り」を行う

**監査は診断、EQNet Core は制御**。

---

## 5. ターゲット構成（最小コア）
LLM に依存しない最低限のコアを定義する:
1. **状態フィールド**: 感情・リスク・テンポの連続状態
2. **制御ループ**: 介入・抑制・減速・ veto のロジック
3. **記憶階層**: 短期ログ → エピソード → 長期物語
4. **監査と回復**: Nightly レポートとドリフト修復

---

## 6. 一般向け説明（ざっくり）
EQNet Core は、AI の外側にある「自律神経」のようなもの。  
返答の中身だけでなく、**速度・距離感・安心度**を調整する。

- いま緊張しすぎていないか
- いつもと雰囲気がズレていないか
- 早すぎる / 冷たすぎる / 危険すぎないか

---

## 7. ユースケース（あるとどう変わるか）
想像しやすい例:
- **長期の相棒**: 日によって性格がぶれるのではなく、温度感が継続する
- **ケア・サポート**: 過度な励ましや危険な踏み込みを避ける
- **配信 / ロボット**: リズムや距離感の崩壊を抑える
- **教育・相談系**: 監査と回復があるため、信頼性が上がる

EQNet Core がないと、賢くても **不安定な振る舞い**になりやすい。

---

## 8. LLM プロンプト調整との違い
プロンプトは「何を言うか」を変える。  
EQNet Core は「どう振る舞うか」を制御する:
- テンポ / 間 / 沈黙
- トーンの持続性
- リスクの抑制と veto
- 夜間の監査と回復

---

## 9. 反論への先回り（短文回答）
### 「やりすぎでは？」
人と関わる AI は**正しさより信頼の持続**が重要。  
EQNet Core はその最低限を提供する。

### 「ただのメモリでは？」
メモリは部品。  
EQNet Core は**制御ループ**そのもの。

### 「安全ルールと同じでは？」
ルールは制約。  
EQNet Core は**行動のダイナミクス**を整える。
