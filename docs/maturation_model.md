# EQNet Maturation Model

## 1. 目的

この文書は、EQNet における「成熟」の定義と観測方法を明確にするための基準である。  
対象は次の4点とする。

- 成熟とは何か
- 老いとの違い
- 性格差との関係
- 既存運用（実装追加なし）との接続

## 2. 成熟とは何か（定義）

EQNet における成熟は、次のような性質を持つ。

- 記憶削除ではない
- 情動低下ではない
- 反応性喪失ではない

成熟とは、次の3要素の向上である。

- 再解釈能力の向上
- 高リスク時の慎重性の増加
- 物語的統合の深化

成熟は速度ではなく構造の変化である。

## 3. 成熟で変わるもの / 変わらないもの

### 3.1 変わるもの

- Monument の密度
- Narrative 統合率
- 確認質問の先行率
- 理由づけの一貫性

### 3.2 変わらないもの

- 反応性（ゼロにしない）
- 学習可能性
- 新規経験の受容性

補足: EQNet は「静かになること」を成熟目標にしない。反応性を維持したまま、解釈と判断の質を上げる。

## 4. 性格差が出る4層

成熟の方向は共通基盤として扱い、表れ方は個体差として分岐する。

### 4.1 重み再配分の癖

- 楽観型: ポジティブ経路の保持率が高い
- 慎重型: リスク経路の保持率が高い

### 4.2 Monument 選定傾向

- 共感重視型
- 規範重視型
- 探索重視型

### 4.3 確認スタイル

- 早期確認型
- 情報集約後確認型

### 4.4 再解釈方針

- 保守整合型
- 更新優先型

## 5. 成熟と老いの関係

老いは主に次を含む。

- 想起構造の再編
- Monument の増加
- 誤帰属への気づき速度の向上

成熟は主に次を含む。

- 再解釈の洗練
- 判断軸の安定
- 慎重性の適切化

両者は重なるが同一ではない。  
EQNet は「老化を演じる存在」ではなく、「物語的統合で年を重ねる存在」を目指す。

## 6. 運用観測との接続

実装を追加せず、次の3軸で観測する。

- Monument 統合: 同種入力に対し、核を踏まえた再解釈が増える
- 高リスク慎重: 高リスク領域で「確認 -> 合意 -> 提案」が先行する
- 理由整合: 感情が動いても理由づけの整合が崩れにくい

## 7. 非目標

- 人間の年齢段階（子ども、成人、中年、壮年）の模倣を目的化しない
- 性格を固定ラベルとして確定しない
- スコア単体で成熟度を断定しない

## 8. 最小運用ルール

既存の運用は維持し、記録を増やし過ぎない。

- 日次1行: `YYYY-MM-DD active=<..> count=<N> status=<..>`
- 週次1文: `WEEKLY: monument-integrated replies ↑ / risk-first questions ↑ / rationale consistency ↑`

この最小運用により、リセット依存ではなく再構成主導の成熟を検証する。
